Рекурсия — это метод программирования, при котором функция вызывает саму себя для решения задачи путем разбиения её на более простые подзадачи.
Это можно сравнить с эффектом бесконечного коридора из отражений в двух зеркалах, стоящих друг напротив друга, или с матрешкой, где внутри большой задачи находится такая же, но меньшего размера.

Для корректной работы рекурсивной функции необходимы два обязательных компонента:

    Базовый случай (base case) — это условие остановки рекурсии, при котором функция может вернуть результат напрямую, без дополнительных рекурсивных вызовов.
    Базовый случай является критически важным, поскольку он предотвращает бесконечную рекурсию и гарантирует корректное завершение алгоритма.

    Рекурсивный шаг (recursive step) — это часть функции, где происходит вызов самой себя, но с измененными параметрами.
    При этом каждый новый вызов должен приближать решение к базовому случаю. Важно, чтобы аргументы при каждом рекурсивном вызове менялись таким образом, чтобы задача постепенно упрощалась.
